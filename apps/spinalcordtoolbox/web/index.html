SCT segmentation in the browser

Segment spinal cord MRI locally in your browser.

Load NIfTI or DICOM spinal cord MRI, run browser-compatible SCT segmentation tasks, inspect overlays, and download outputs without uploading patient data.

Patient images, DICOM metadata, voxel values, segmentations, screenshots, and generated outputs stay on this computer.

Local-first workflow

  • Files are read into browser memory for viewing and analysis.
  • Public runtime and model assets may be downloaded when needed.
  • Patient-derived files and generated results are not uploaded by the app.
  • Long runs can continue if this site is allowed to stay active in your browser.

How It Works

1. Load MRI data

Drop NIfTI or DICOM files. Loading only displays the scan; processing starts when you run a task.

2. Choose an SCT task

Select a supported segmentation task and adjust threshold or cleanup settings if needed.

3. Inspect and export

Review overlays in NiiVue, toggle label stages, download NIfTI outputs, and run supported post-processing.

About SpinalCordToolbox

      
    

    

      

        

          SpinalCordToolbox
          
Spinal cord MRI segmentation in the browser

        

      


      
SpinalCordToolbox is a browser-based tool for SCT stable spinal cord MRI segmentation workflows.

      
All image computation runs locally in your browser using ONNX Runtime Web. Patient image data is not uploaded to any server.


      
Version

      

        

          SpinalCordToolbox
          
        

      

    

  







  

    

      
Privacy

      
    

    

      

        
        

          Your data never leaves your device.
          
SpinalCordToolbox runs entirely in your web browser using WebAssembly and ONNX Runtime. All processing occurs locally on your machine.

        

      


      
What this means

      
    
        
  • No patient image data, metadata, masks, segmentations, or outputs are uploaded to any server or cloud service
    • 
        
  • No internet connection is required after the page loads
    • 
        
  • Sensitive patient data remains on your computer at all times
    • 
        
  • There is no need to anonymise or de-identify your data before use
    • 
      


      
How it works

      
SpinalCordToolbox uses ONNX Runtime Web to run browser-compatible deep learning models directly in the browser via WebAssembly. Your NIfTI or DICOM files are read into browser memory, processed locally, and results are displayed without patient-data network transfer.


      
Third-party services

      
Google Analytics collects aggregate page usage and performance metrics when your browser's Do Not Track setting is not enabled. Those statistics must not include patient image data, DICOM metadata, filenames, voxel values, masks, segmentations, screenshots, free-text patient-derived logs, or downloaded outputs.
