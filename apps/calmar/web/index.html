Lesion mapping and reporting in the browser

Process stroke lesion maps locally in your browser.

CALMaR guides a structural T1 through brain extraction, editable lesion-mask review, MNI registration, atlas overlap, functional-connectivity mapping, and reportable tables.

Patient images, masks, voxel values, screenshots, and generated outputs stay on this computer.

Local-first workflow

  • Files are read into browser memory for viewing and analysis.
  • Atlas and model assets may be downloaded when a workflow needs them.
  • Patient-derived files and generated results are not uploaded by the app.
  • Background execution is possible if this site is added under "Always keep these sites active" in your browser settings.

How It Works

1. Load T1 data

Choose a structural T1. Loading only displays the scan; processing starts when you run analysis.

2. Review the mask

Use an automatic seed, blank manual mask, or uploaded mask, then edit and confirm before downstream analysis continues.

3. Map and report

Register to MNI, compute selected-atlas overlap, generate a connectivity map, and export CSV/NIfTI outputs locally.

About CALMaR

CALMaR

Co-designed Automated Lesion Mapping and Reporting.

This app loads a structural T1 (or a manual MNI-aligned lesion mask) locally, runs the CALMaR lesion mapping chain in the browser, and never uploads patient data.

The pipeline: SynthStrip brain extraction → SynthStroke baseline lesion segmentation → SynthMorph deformable registration to MNI152 (1603 1mm) → affine-aware resample to the selected atlas grid → atlas overlap → group-FC weighted-sum → threshold + cluster cleanup. All stages are exposed as individual buttons; "Run full pipeline" chains them in one click.

Privacy

Your data remains on your device.

Uploaded images and masks are read into browser memory for local viewing and analysis.

What this means

  • No patient image data, lesion masks, voxel values, screenshots, or generated outputs are uploaded by the app.
  • Atlas and model assets may be fetched by the browser when a workflow needs them.
  • Downloaded result files are created locally from browser memory.

Third-party services

Static app files and public atlas assets may be served from external hosting, but patient-derived files are not sent to those services by the analysis code. Google Analytics collects aggregate page usage and performance metrics without patient images, masks, voxel values, screenshots, generated outputs, filenames, or analysis results. If your browser's Do Not Track setting is enabled, Google Analytics tracking is not configured.

Brain extraction

SynthStrip brain extraction model

CALMaR uses manifest asset lnm-synthstrip, runtime file models/synthstrip.onnx, source version SynthStrip-Hoopes2022-Apache-2.0. The ONNX asset was converted from FreeSurfer's official main SynthStrip model v1 checkpoint, synthstrip.1.pt; it is not the no-CSF or pediatric variant. The browser runs the ONNX FP32 export through the fast WASM wrapper with no final dilation.

Hoopes A, Mora JS, Dalca AV, Fischl B, Hoffmann M. SynthStrip: skull-stripping for any brain image. NeuroImage. 2022;260:119474.

https://surfer.nmr.mgh.harvard.edu/docs/synthstrip/ Original FreeSurfer checkpoint Hosted ONNX asset

Lesion segmentation

SynthStroke baseline lesion model

CALMaR uses manifest asset lnm-stroke-lesion, runtime file models/lnm-stroke-lesion.onnx, source version SynthStroke-baseline-MELBA2025-MIT. The ONNX asset was converted from liamchalcroft/synthstroke-baseline config.json and model.safetensors, not SynthStroke SynthPlus. The browser runs 1283 sliding-window patches with no TTA, then collapses background/stroke logits to stroke probability before thresholding.

Chalcroft L, Pereira S, Reinke A, et al. SynthStroke: synthetic chronic stroke lesion segmentation models. MELBA. 2025.
