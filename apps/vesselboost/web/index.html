Vessel segmentation and review in the browser

Process brain vessel maps locally in your browser.

VesselBoost guides NIfTI or DICOM angiography data through optional preprocessing, 3D UNet vessel segmentation, brain masking, interactive review, and reportable local outputs.

Patient images, masks, voxel values, screenshots, and generated outputs stay on this computer.

Local-first workflow

  • Files are read into browser memory for viewing and analysis.
  • Model, runtime, and preprocessing assets may be downloaded when a workflow needs them.
  • Patient-derived files and generated results are not uploaded by the app.
  • Background execution is possible if this site is added under "Always keep these sites active" in your browser settings.

About VesselBoost

VesselBoost

Blood vessel segmentation in the browser

VesselBoost is a browser-based tool for automatic segmentation of blood vessels from MRI angiography using a 3D UNet model.

Image processing runs locally in your browser using ONNX Runtime Web. Patient image data and generated outputs are not uploaded by the analysis code.

Version

VesselBoost

Privacy

Your patient data never leaves your device.

VesselBoost image processing runs in your web browser using WebAssembly and ONNX Runtime. Patient-derived files stay local to your machine.

What this means

  • Patient images, voxel values, screenshots, and generated outputs are not uploaded by the app
  • Model, runtime, and preprocessing assets may be fetched by the browser when needed
  • Sensitive patient data remains on your computer during analysis
  • There is no need to anonymise or de-identify your data before use

How it works

VesselBoost uses ONNX Runtime Web to run deep learning models directly in the browser via WebAssembly. Your NIfTI or DICOM files are read into browser memory, processed locally, and results are displayed without network transfer of patient data.

Third-party services

Static app files and public model assets may be served from external hosting, but patient-derived files are not sent to those services by the analysis code. Google Analytics collects page usage metrics without patient images, voxel values, screenshots, generated outputs, filenames, or analysis results. Analytics tracking is disabled when your browser sends a Do Not Track value of "1" or "yes".
